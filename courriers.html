Projet : Gestion des Courriers

Application web pour gérer les courriers Départ et Arrivée

Présentation du projet

Ce projet consiste à développer une application web permettant de gérer efficacement les courriers dans un environnement professionnel. L'application offre des fonctionnalités adaptées aux besoins des utilisateurs, selon leur rôle et le type de courrier.

Fonctionnalités principales

  • Gestion des courriers Arrivée et Départ.
  • Ajout, modification, suppression et archivage des courriers.
  • Affectation des courriers aux services et bureaux.
  • Page d'accueil personnalisée selon le rôle de l'utilisateur.
  • Tableaux interactifs avec sélection multiple et boutons d’action.
  • Formulaires dynamiques avec validation et date automatique.
  • Navigation fluide entre les sections : Accueil, Courriers, Archives, Référentiel.

Technologies utilisées

Node.js Express EJS HTML CSS JavaScript MySQL / SQLite

Organisation du projet

Le projet est structuré en plusieurs dossiers pour séparer les responsabilités :

  • controllers/ : logique de traitement des requêtes.
  • models/ : gestion de la base de données.
  • views/ : pages EJS affichées aux utilisateurs.
  • public/ : fichiers CSS, JS et images.
  • database/ : scripts et fichiers de base de données.

CRUD et gestion des données

L’application implémente le concept CRUD (Create, Read, Update, Delete) pour toutes les entités :

  • Créer des courriers et des utilisateurs.
  • Afficher les listes de courriers et utilisateurs.
  • Modifier les informations et réaffecter les courriers.
  • Supprimer ou archiver les éléments selon les besoins.

Interface utilisateur

L’interface est simple et intuitive :

  • Tableaux interactifs pour visualiser les courriers.
  • Boutons d’action selon le rôle de l’utilisateur.
  • Formulaires dynamiques et modals pour gérer les données en temps réel.
